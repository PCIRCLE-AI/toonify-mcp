Best for Large Tool Output

Toonify is most useful when your token cost comes from structured data, generated output, or source files, not from normal chat.

BEST FIT

Large structured payloads

Teams reading large JSON, CSV, YAML, API responses, and generated data into Claude Code.

ALSO USEFUL

Code-heavy sessions

TypeScript, Python, Go, and PHP source files where comments and whitespace add token cost without helping the model much.

NOT IDEAL

Small or prose-first tasks

If your context is mostly short prose, very small files, or content that depends heavily on original formatting, the gains are usually limited.

See the Difference in One Glance

The goal is not different information. The goal is fewer tokens for the same working session.

Before Optimization (142 tokens)

{ "products": [ {"id": 101, "name": "Laptop Pro", "price": 1299}, {"id": 102, "name": "Magic Mouse", "price": 79} ] }

After Optimization (57 tokens, -60%)

[TOON-JSON] products[2]{id,name,price}: 101,Laptop Pro,1299 102,Magic Mouse,79
Automatically applied in Plugin mode - no manual calls needed.

Start With the Default Path

Plugin mode is the fastest way to try Toonify. Use MCP server mode only when you need explicit control.

Recommended: Plugin Mode

Best for Claude Code users who want automatic optimization with no manual tool calls.

# Download the repository git clone https://github.com/PCIRCLE-AI/toonify-mcp.git cd toonify-mcp # Install deps and build npm install npm run build # Install globally from local source npm install -g . # Add as plugin (automatic mode) claude plugin add toonify-mcp # Verify installation claude plugin list # Should show: toonify-mcp ✓

Result: supported structured data and source-code results are optimized automatically after tool use.

Advanced: MCP Server Mode

Use this when you want explicit control or need to connect Toonify to another MCP client.

# Prerequisite: complete plugin install so toonify-mcp is available # Register as MCP server claude mcp add toonify -- toonify-mcp # Verify claude mcp list # Should show: toonify: toonify-mcp - ✓ Connected

For explicit control or non-Claude Code MCP clients. Call tools manually when needed.

Why Teams Use Toonify

The value is simple: less token-heavy output, with less workflow friction.

🎯

Automatic in Plugin Mode

PostToolUse hook intercepts supported tool results and applies optimization automatically, so the default experience stays hands-off.

claude plugin add toonify-mcp
🧭

Workflow Stays the Same

The pitch is not a new interface. It is lower token overhead inside the workflow your team already uses.

📦

Large Structured Payloads

Built for large JSON, CSV, YAML, API responses, and generated output that would otherwise bloat context.

🌍

Supported Code Compression

Supports TypeScript, Python, Go, and PHP source-code compression through comment and whitespace reduction in the pipeline.

🔄

Dual Mode

Works as Plugin (automatic) or MCP Server (manual). Choose the mode that fits your workflow.

⚖️

Selective Optimization

Structured data is compressed only when estimated savings clear the configured threshold.

📊

Local Metrics and Safe Fallback

Track savings locally, and if optimization is not worth it, Toonify falls back to the original content.

How It Works

A short version of the default plugin flow

1

Read a large output

Claude Code fetches a large JSON file, CSV, YAML response, or supported source file.

2

Intercept the tool result

The PostToolUse hook checks whether the returned content is worth compressing.

3

Compress when it helps

Structured data is converted to compact TOON format. Supported code is reduced through comment and whitespace compression.

4

Continue the same session

Optimized content moves forward automatically when estimated savings clear the configured threshold.
